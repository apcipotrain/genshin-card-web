## 1. 页面系统

### 1.1 路由架构

**文件**: `src/ui/router.ts`

```typescript
export type PageId = 'home' | 'chapters' | 'match' | 'waiting' | 'game' | 'result' | 'login';

interface RouteState {
  page: PageId;
  params?: Record<string, unknown>;  // 路由参数
}
```

**流转关系**:

```
Login → Home → Chapters → Game(PVE) → Result → Home
              → Match → Waiting → Game(PVP) → Result → Home
```

### 1.2 页面与参数传递

| 源页面 | 目标页面 | 传递参数 | 说明 |
|--------|----------|----------|------|
| Login | Home | 无 | 登录成功后 `socketManager.setAccount()` |
| Home | Chapters | 无 | PVE 入口 |
| Chapters | Game | `{ levelId: number }` | 关卡 ID |
| Home | Match | 无 | PVP 入口 |
| Match | Waiting | `{ roomId: string, isHost: boolean, mySlotIndex: number }` | 房间信息 |
| Waiting | Game | 无 | 由 `game_start` 事件触发 |
| Game | Result | `{ winner: string }` | 胜负结果 |

### 1.3 GamePage 关键属性

**文件**: `src/ui/GamePage.ts`

```typescript
class GamePage {

  // 核心组件
```

### 1.4 页面与其他子系统的关系

```
GamePage
  ├── 页面 ← → 卡牌:  players[].handCards / equipZone / judgeZone
  ├── 页面 ← → AI:    HumanWebUIDriver (人类) + DelayedAIDriver (AI)
  ├── 页面 ← → 座位:  getBattlefieldLayout() / getPVEBattlefieldLayout()
  ├── 页面 ← → BGM:   switchBGMForMonarch() / getBgmForRegion()
  └── 页面 ← → 等级:  showResultModal() → computeLocalPVPExp() → add_exp
```

## 2. 卡牌系统

### 2.1 数据结构

**文件**: `src/core/types.ts`, `src/data/CardData.ts`

```typescript
interface Card {
  id: number;
  name: string;
  type: CardType;           // Basic | Magic | Equipment
  suit: SuitType;           // Spade | Heart | Club | Diamond
  number: number;           // 1-13
  description: string;
  element?: ElementType;    // 属性杀: Pyro | Hydro | Cryo | Electro | Anemo | Geo | Dendro
  equipType?: EquipmentType;
  weaponRange?: number;
  cardSource?: PlayerState; // 打出此牌的玩家引用
  isVirtual?: boolean;      // 是否为虚拟牌（技能生成）
}
```

### 2.2 卡牌流转链路

```
CardData.ts (160张静态数据)
    ↓
DeckManager.init(CARD_DATA)  → 洗牌入摸牌堆
    ↓
GameFlowController.drawPhase() → 玩家摸牌 → PlayerState.handCards[]
    ↓
HumanWebUIDriver.promptPlayCard() → 玩家选择手牌索引
    ↓
CardEffectManager.handleActivePlay() → 执行卡牌效果
    ↓
DeckManager.discard() → 弃牌堆 / returnToDrawPile() → 回摸牌堆
```

### 2.3 卡牌与页面的传递

| 场景 | 数据流向 | 涉及方法 |
|------|----------|----------|
| 渲染手牌 | `PlayerState.handCards[]` → DOM | `GamePage.renderHandCards()` |
| 出牌选择 | DOM 点击 → `handCards[idx]` | `HumanWebUIDriver.promptPlayCard()` |
| 出牌动画 | `Card` 对象 → 飞行动画 | `GamePage.animateCardPlayed()` |
| 装备显示 | `PlayerState.equipZone` → DOM | `GamePage.renderEquipDisplay()` |
| 判定区显示 | `PlayerState.judgeZone[]` → DOM | `GamePage.renderSeatCard()` |
| PVP 出牌 | 服务端 `Card` → JSON → 客户端渲染 | `processPVPSingleEvent(CardResponded)` |

### 2.4 PVP 卡牌序列化

服务端发送的 `game_event` 中包含 `players` 数组，每个 player 的 `handCards` 对其他玩家**不可见**（服务端 `sanitizePlayerForViewer` 将非自己的玩家 `handCards` 清空为 `[]`，仅保留数量）。

## 3. AI 层

### 3.1 架构

```
IPlayerDriver (接口)
    ├── HumanWebUIDriver  — 人类玩家（PVE/PVP共用）
    ├── AIDriver          — AI 决策引擎
    └── DelayedAIDriver   — 延迟包装器（包装 AIDriver）
    └── RemotePlayerDriver — 远程玩家（PVP服务端，通过Socket通信）
```

**文件分布**:
- `src/ai/AIDriver.ts` — AI 决策逻辑
- `src/ai/DelayedAIDriver.ts` — 延迟包装
- `src/ui/GamePage.ts` 内 `HumanWebUIDriver` — 人类玩家交互
- `server/RemotePlayerDriver.ts` — PVP 远程驱动

### 3.2 IPlayerDriver 接口

**文件**: `src/core/types.ts`

```typescript
interface IPlayerDriver {
  promptPlayCard(state, ctx): Promise<number>;        // 出牌选择（返回手牌索引，-1=跳过）
  promptTarget(state, validTargets, reason, ctx): Promise<number | null>;  // 选择目标
  promptResponse(state, cardName, ctx): Promise<number>;  // 响应索要（闪/杀/桃等）
  promptDiscard(state, ctx): Promise<number>;         // 弃单张

  promptZone(state, targetId, ctx): Promise<ZoneSelection>;  // 选择区域
  promptNullification(state, ctx): Promise<boolean>;  // 无懈可击
  promptIronChainMode?(state, ctx): Promise<number>;  // 铁索连环模式
  promptAmazingGrace?(tableCards, ctx): Promise<number>;  // 五谷丰登选牌
  promptSelectCard?(state, title, filter, ctx): Promise<number>;  // 通用选牌
  promptYesNo?(question): Promise<boolean>;          // 是/否
  promptShowCard?(state, ctx): Promise<number>;      // 火攻展示
  promptRansackHand?(state, targetId, ctx): Promise<number>;  // 顺手牵羊选牌
  promptGenderWeapon?(state, attackerName, ctx): Promise<number>;  // 雌雄双股剑
  promptZhanBa?(state, ctx): Promise<number[]>;       // 丈八蛇矛选两张
  // ... 更多 prompt 方法
}
```

### 3.3 AI 决策流程

```
AIDriver.promptPlayCard(state, ctx)
    ↓
sortCardsByPriority(me, ctx)
    ↓
scoreCard(card, me, ctx, enemies, allies)  → 评分（0-100）
    ↓
返回最高分牌索引
```

**评分规则** (`AIDriver.scoreCard`):

| 卡牌 | 分值 | 条件 |
|------|------|------|
| 装备牌 | 90 | 无条件 |
| 桃 | 100/0 | HP < maxHP / 满 |
| 酒 | 75/80/30 | 有杀/HP≤1/其他 |
| 无中生有 | 95 | 无条件 |
| 顺手牵羊 | 85/0 | 距离1内有目标 / 否则 |
| 过河拆桥 | 80/0 | 有目标 / 否则 |
| 杀 | 70 | 有敌人 |
| 南蛮/万箭 | 60 | 有敌人 |
| 决斗 | 75/50 | 弱敌 / 一般 |
| ... | ... | ... |

### 3.4 AI 与其他子系统的关系

| 子系统 | 关系 |
|--------|------|
| **卡牌** | AI 读取 `PlayerState.handCards` 决策；`scoreCard` 根据卡牌类型/名称评分 |
| **页面** | PVE: `HumanWebUIDriver` + `DelayedAIDriver` 本地驱动；PVP: `RemotePlayerDriver` 通过 Socket 通信 |
| **座位** | AI 通过 `ctx.players` 获取所有玩家，按 `faction`(PVE) 或 `role`(PVP) 判断敌友 |
| **等级** | 无直接关系（AI 不获取经验） |

### 3.5 PVP 模式 AI 替代

- **断线时**: `GameHost.handleDisconnect` 将真人 `RemotePlayerDriver` 替换为 `DelayedAIDriver(pid, () => 1200)`
- **填充位**: 房间 `aiFill` 数量的 AI 玩家始终使用 `DelayedAIDriver`
- **延迟**: PVE 可调 0.3s-5.0s；PVP 固定 1200ms

## 4. 座位系统

### 4.1 PVP 座位布局

**文件**: `src/ui/GamePage.ts` — `getBattlefieldLayout()`

PVP 模式（8人固定），人类玩家在 `humanPlayerIdx`（通常为 1）：

```typescript
// 逆时针布局：self(右下) → right(下往上) → top(右到左) → left(上往下)
{
  right: [p[(hIdx+2)%8], p[(hIdx+1)%8]],           // 下→上
  top:   [p[(hIdx+5)%8], p[(hIdx+4)%8], p[(hIdx+3)%8]],  // 左→右（逆时针远→近）
  left:  [p[(hIdx+6)%8], p[(hIdx+7)%8]],           // 上→下
  self:  p[hIdx],
}
```

**视觉示意**:
```
  top[2]=+5  top[1]=+4  top[0]=+3
left[0]=+6                right[1]=+2
left[1]=+7                right[0]=+1
              self=hIdx
```

### 4.2 PVE 座位布局

**文件**: `src/ui/GamePage.ts` — `getPVESeatLayout()` + `getPVEBattlefieldLayout()`

PVE 使用 `pveSeatIndex` 标记座位，人类玩家固定在 seatIndex=0：

```typescript
getPVESeatLayout(totalPlayers): number[]
// 8人: [0,1,2,3,4,5,6,7]
// 6人: [0,1,2,3,5,7]     (跳过4,6)
// 4人: [0,3,5,7]          (跳过1,2,4,6)
// ...
```

渲染时与 PVP 相同的逆时针布局规则，以人类玩家座位（0）为起点。

### 4.3 PVP 服务端座位

**文件**: `server/GameHost.ts`

```
room.slots[0..7] → players[0..7]（按槽位索引严格对应）
```

- `playerSlotMap`: `playerId → { slotIndex, socketId, token, account, isAI }`
- 服务端 `players[i]` 严格对应 `room.slots[i]`，确保座位布局与 UI 一致
- 主公由选将阶段确定，通过 `setMonarchPlayerId(monarchSlotIndex)` 设置

### 4.4 座位与其他子系统的关系

| 子系统 | 关系 |
|--------|------|
| **页面** | `getBattlefieldLayout()` 读取 `players[]` 按 `humanPlayerIdx` 排列到 top/left/right/self |
| **卡牌** | 距离计算 `DistanceCalc.getDistance()` 依赖座位索引计算邻居关系 |
| **AI** | AI 通过 `ctx.players` 数组索引判断敌友（PVE: `faction`，PVP: `role`） |
| **BGM** | PVP: 根据**主公**的 `region` 切换；PVE: 根据**章节**的 `region` 切换 |
| **等级** | 无直接关系 |

### 5.2 切换逻辑

#### PVE 模式

```

    ↓

    ↓

    ↓

```

**关键**: PVE 使用**章节地区**（`chapter.region`），而非英雄地区。

#### PVP 模式

```

    ↓

    ↓

       || players.find(p => p.role === Monarch)     // PVP
       || players[0]
    ↓
region = anchor.region
    ↓
wallpaperRegionBase + BGM 路径
```

**关键**: PVP 使用**主公的武将地区**（`monarchPlayer.region`）。

## 6. 等级系统

### 6.1 数据结构

**文件**: `server/AccountManager.ts`

```typescript
interface Account {
  id: string;
  name: string;
  passwordHash: string;
  nickname: string;
  avatar: string;
  level: number;      // 当前等级 (1-60)
  exp: number;        // 累计经验总量
  createdAt: number;
}
```

### 6.2 经验计算

**服务端**: `AccountManager.getLevelAndProgress(exp)`

```typescript
// 累计经验表 (CUMULATIVE)
// Lv.1-55: 每级需要 lv*100 经验
// Lv.56: 56000, Lv.57: 57000, Lv.58: 58000, Lv.59: 59000, Lv.60: 封顶

function getLevelAndProgress(exp: number): {
  level: number;          // 当前等级
  currentExp: number;    // 当前等级内已有经验 (= exp - 该等级累计阈值)
  nextExp: number;       // 升下一级所需总经验 (该等级的升级需求)
}
```

**客户端**: `SocketManager.ts` 内有完全相同的 `buildCumulative()` + `getLevelAndProgress()` 函数。

### 6.3 经验来源与流转

#### PVP 结算

```
GameHost.computeExpForAllPlayers()
    ↓
对每个真人玩家计算:
  baseExp = GameHost.calcBaseExp(role, winner)
    // 胜方: 主公75/忠臣50/反贼40/内奸100
    // 败方: 20
  bonusExp = GameHost.calcKillExp(role, killStats)
    // 按击杀目标身份加成
  totalExp = baseExp + bonusExp
    ↓
accountManager.addExpByAccountId(accountId, totalExp)
    ↓
game_over 事件下发 expByPlayerId: {
  [playerId]: { baseExp, bonusExp, totalExp, oldLevel, newLevel, leveledUp, escaped }
}
    ↓
客户端 showResultModal() 显示
  "获得经验" = baseExp + bonusExp  (本局获得量)
```

#### PVE 结算

```
GamePage.computeLocalPVPExp(winner)
    ↓
本地计算经验
    ↓
socketManager.emitWithAck('add_exp', { totalExp })
    ↓
服务端 accountManager.addExp(token, totalExp)
    ↓
返回 { oldLevel, newLevel, totalExp, leveledUp }
```

### 6.4 客户端显示

**文件**: `src/network/SocketManager.ts` — `setAccount()`

```typescript
setAccount(account) {
  // 如果 account.exp 存在但 currentLevelExp/nextLevelExp 未设置
  // 自动从 exp 计算（与服务端逻辑一致）
  if (account.exp !== undefined && (currentLevelExp === undefined || nextLevelExp === undefined)) {
    const progress = getLevelAndProgress(account.exp);
    account.currentLevelExp = progress.currentExp;
    account.nextLevelExp = progress.nextExp;
    if (account.level === undefined) account.level = progress.level;
  }
}
```

**显示链路**:
```
服务端 profile_data 事件 → { account, progress }
    ↓
HomePage.show() → socketManager.on('profile_data')
    ↓
socketManager.setAccount(data.account)  // 自动计算 currentLevelExp/nextLevelExp
    ↓
renderPanelContent() 显示 "currentLevelExp / nextLevelExp"
```

## 7. 子系统间传递关系总览

### 7.1 完整传递图

```
┌─────────────┐     levelId      ┌──────────────┐
│   页面系统    │ ──────────────→ │   卡牌系统    │
│  GamePage    │                 │ DeckManager  │
│              │ ← handCards ─── │ CardEffect   │
└──────┬───────┘                 └──────┬───────┘
       │                                │
       │ players[]                      │ ctx.players
       │                                │
       ▼                                ▼
┌─────────────┐     IPlayerDriver ┌──────────────┐
│   座位系统    │ ←────────────── │    AI 层      │
│ getBattle-   │                 │ AIDriver      │
│ fieldLayout  │                 │ HumanWebUI    │
└──────┬───────┘                 │ RemoteDriver  │
       │                          └──────────────┘

```

### 7.2 关键数据流摘要

| 数据 | 来源 | 消费方 | 传递方式 |
|------|------|--------|----------|
| `PlayerState[]` | GameFlowController | GamePage / AIDriver | 内存引用 (PVE) / JSON序列化 (PVP) |
| `Card` 对象 | DeckManager | CardEffectManager → UI | `PlayerState.handCards[]` |
| `ctx: GameContextSnapshot` | GameFlowController | IPlayerDriver.promptXxx() | 函数参数 |
| `region` (地区) | PlayerState.region / chapter.region | GamePage.getBgmForRegion() | 属性读取 |
| `exp` (经验) | GameHost.computeExp / add_exp 事件 | AccountManager / SocketManager | Socket 事件 |
| `pveSeatIndex` | makePVEPlayer() | getPVEBattlefieldLayout() | PlayerState 属性 |
| `turnOrder` | PVELevel.turnOrder | GameFlowController.startGame() | 参数传递 |
| `faction` | makePVEPlayer() | AI / checkGameOver | PlayerState 属性 |
| `role` | assignRoles / setMonarchPlayerId | AI / sanitizePlayerForViewer | PlayerState 属性 |

### 7.3 PVP 模式完整链路

```
1. 客户端: socketManager.emit('login', {name, password})
2. 服务端: accountManager.login() → 返回 {account, token}
3. 客户端: socketManager.setAccount(account)  // 自动计算 level/exp
4. 客户端: socketManager.emit('create_room', {roomName, password})
5. 服务端: roomManager.createRoom() → 返回 {roomId, room}
6. 客户端: router.navigate('waiting', {roomId})
7. 客户端: socketManager.emit('toggle_ready', {ready: true})
8. 房主: socketManager.emit('start_hero_select')
9. 服务端: 随机选主公 → emit('hero_select_start'/'hero_select_waiting')
10. 客户端: 选将 → socketManager.emit('select_hero', {heroId})
11. 服务端: checkAllSelectedAndStart() → new GameHost() → host.initGame()
12. 服务端: emit('game_start', {players, yourPlayerId, drawPileCount})
13. 客户端: handlePVPGameStart() → 渲染战场 → switchBGMForMonarch()
14. 服务端: host.start() → GameFlowController 游戏循环
15. 服务端: emit('game_event', {type, data:{..., players, drawPileCount}})
16. 客户端: processPVPSingleEvent() → 渲染
17. 服务端: RemotePlayerDriver.request() → emit('prompt', {type, requestId, ...})
18. 客户端: handlePVPPrompt() → HumanWebUIDriver → 用户点击 → socketManager.respond()
19. 服务端: game_over → computeExpForAllPlayers() → accountManager.addExpByAccountId()
20. 服务端: emit('game_over', {winner, killStats, expByPlayerId, playerRoles})
21. 客户端: showResultModal() → 显示经验变化 → 返回主页
```

### 7.4 PVE 模式完整链路

```
1. ChaptersPage: 选择关卡 → router.navigate('game', {levelId})
2. GamePage.show(): 加载 PVELevel → showPVEHeroSelect()
3. 选将完成: finishPVEHeroSelect()
   → getPVESeatLayout(totalPlayers) → 分配座位
   → makePVEPlayer() 创建 PlayerState (含 faction, pveSeatIndex)
   → 章节级 BGM/背景设置 (chapter.region)
4. initGameCore(): EventBus + DeckManager + CardEffectManager + ...
   → HumanWebUIDriver (人类) + DelayedAIDriver (AI)
5. startGame(): flowController.startGame(humanId, turnOrder)
6. 游戏循环: EventBus 事件 → GamePage 渲染
7. 游戏结束: GameOver 事件
   → computeAndSavePVEStars() (星级)
   → computeLocalPVPExp(winner) (本地经验)
   → showResultModal()
   → socketManager.emitWithAck('add_exp', {totalExp}) (上报服务端)
   → 返回章节选择
```
