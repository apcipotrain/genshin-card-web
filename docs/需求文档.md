# 原神杀（Genshin Card）需求文档

> 版本：v2.0 | 日期：2026-06-18 | 状态：PVE+PVP联机完成 + 29武将技能 + 账号系统 + 经验等级

---

## 一、项目概述

### 1.1 项目定位

**原神杀**是一款以《三国杀》军争八人场为核心玩法底层、以《原神》角色为武将的网页卡牌游戏。

### 1.2 目标平台

- **Web 网页端**（PC 浏览器优先，后续适配移动端）
- 技术栈：TypeScript + Vite + 原生 DOM 渲染 + Socket.IO

### 1.3 核心玩法

- **PVE 闯关模式**：玩家控制武将与人机对战，无身份机制，仅分敌友（2-8人动态布局）
- **PVP 联机模式**：军争八人场，标准身份局（主公/忠臣×2/反贼×4/内奸），支持真人+AI混合

## 二、功能需求

### 2.1 页面结构（7个核心页面）

| 页面 | 名称 | 描述 |
|------|------|------|
| 1 | 登录页面 | 账号注册/登录，token 认证 |
| 2 | 主页面 | PVE 与 PVP 两大模式入口 + 账号管理 + 图鉴 |
| 3 | 关卡选择 | 大章横排滚动 + 小关弹窗选择 |
| 4 | 匹配界面 | PVP 房间列表 + 创建房间 |
| 5 | 等待界面 | 房间内准备/取消准备 + AI 填充 |
| 6 | 游戏界面 | 8人军争布局 + 手牌区 + 技能区 |
| 7 | 结算界面 | 胜负结果表 + 经验/升级显示 |

### 2.2 登录页面（Login）

- 账号注册（名称 + 密码，SHA256 哈希存储）
- 账号登录（名称 + 密码，返回 token）
- Token 自动认证（刷新页面自动登录）
- 账号持久化：`server/accounts.json`

### 2.3 主页面（Home）

- **左侧卡片**：PVE 闯关模式入口，点击进入关卡选择页面
- **右侧卡片**：PVP 联机模式入口，点击进入匹配界面
- **左侧栏**：账号管理、成就系统、📖图鉴（神将资料库）、设置
- **账号管理弹窗**：显示等级、经验进度（currentLevelExp/nextLevelExp）、昵称修改
- 视觉风格：深色主题 + 金色点缀，原神风格

### 2.4 PVE 关卡选择页面（Chapters）

- **9个大章**横向排列，左右滑动/箭头切换：
  - 蒙德 → 璃月 → 稻妻 → 须弥 → 枫丹 → 纳塔 → 挪德卡莱 → 至冬 → 深境螺旋
- 每章有独立背景色渐变
- 点击大章弹出小关选择弹窗
- **解锁条件**：等级要求 + 前一章节累计 ≥ 20 星
- 当前已实现：蒙德7关 + 璃月12关

### 2.5 PVP 匹配界面（Match）

- **左侧**：已有房间列表（可滚动），显示房间号、人数、密码状态
- **右侧**：创建房间面板
  - 房间名称输入
  - 私密模式开关（开启后可设密码）
  - 快速匹配按钮

### 2.6 房间等待界面（Waiting）

- 显示房间号
- 8个玩家位（2×4网格），区分空位/已占/已准备
- 准备/取消准备按钮
- AI 填充数量设置（房主可调）
- 离开房间按钮

### 2.7 游戏主界面（Game）

**布局方案**：左右切割 3:1

#### 左侧（游戏主区域）：
- **顶部信息栏**：⏱ 游戏计时器、牌堆剩余数、当前阶段、第X轮·第X回合
- **战场布局**（8人逆时针）：
  - 上方 3 人（对手）
  - 左侧 2 人（对手）
  - 右侧 2 人（对手）
  - 下方 1 人（自己）
- **底部玩家区**：
  - 手牌区（横向排列，显示花色点数）
  - 技能区（手牌区内部右侧，最多5个技能按钮，含武将元素属性）

#### 右侧（信息面板）：
- **上方 3/4**：游戏记录（实时滚动日志）
- **下方 1/4**：聊天记录（预留）

### 2.8 结算界面（Result）

- 模态弹窗展示
- 表格列出所有玩家：昵称、身份、武将、体力、存活/阵亡、获得经验
- PVP 模式：显示经验变化（baseExp + bonusExp）、升级动画
- PVE 模式：星级评定（1-3星）
- 关闭按钮 + 返回主页按钮

## 三、游戏机制需求

### 3.1 牌堆

- **160 张标准三国杀牌堆**（`src/data/CardData.ts`）：
  - 基本牌 85 张：杀 44 + 闪 24 + 桃 12 + 酒 5
  - 锦囊牌 50 张
  - 装备牌 26 张

### 3.2 身份机制（PVP）

- 8 人军争：主公×1、忠臣×2、反贼×4、内奸×1
- 主公体力上限+1
- 奖惩：杀反摸3张、主杀忠弃所有牌
- 选将阶段随机指定主公（含AI主公）

### 3.3 阵营机制（PVE）

- 仅分友方（Ally）和敌方（Enemy），无身份系统
- 胜利条件：敌方全员阵亡 + 自己存活
- 失败条件：自己阵亡
- 出牌顺序由关卡 `turnOrder` 指定

### 3.4 回合流程

```
准备 → 判定 → 摸牌 → 出牌 → 弃牌 → 结束
```

- **PVE 模式**：根据关卡所属章节的 `region` 自动切换背景和BGM
- **PVP 模式**：根据主公所属 `region` 自动切换背景和BGM
- **背景壁纸轮播**：每国10张壁纸，每分钟随机切换一张，Fisher-Yates 洗牌

  - 蒙德: 壁纸 10-19 | 璃月: 20-29 | 稻妻: 30-39 | 须弥: 40-49

## 五、经验与等级系统

### 5.1 等级表

| 等级范围 | 升级所需经验 |
|----------|-------------|
| Lv.1-55 | 当前等级 × 100 |
| Lv.56-59 | 56000-59000 |
| Lv.60 | 封顶 |

### 5.2 经验来源

- PVP 胜利：主公75 / 忠臣50 / 反贼40 / 内奸100
- PVP 失败：基础20
- 击杀加成：按击杀目标身份计算 bonus
- PVE：根据关卡难度和星级计算
- 逃跑玩家：经验为0

### 5.3 解锁条件

- 蒙德 Lv.1 | 璃月 Lv.4 | 稻妻 Lv.7 | 须弥 Lv.10
- 枫丹 Lv.13 | 纳塔 Lv.16 | 挪德卡莱 Lv.21 | 至冬 Lv.26 | 深境螺旋 Lv.31
- 前一章节累计 ≥ 20 星

## 六、非功能性需求

- 页面切换动画流畅（0.3s 过渡）
- 支持触摸滑动（章节选择页）
- 响应式布局（优先 PC，后续适配移动端）
- 深色主题保护视力
- 断线自动重连（Socket.IO reconnection）
- 服务端异常捕获（uncaughtException 不退出）
