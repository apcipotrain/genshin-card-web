# PVE 闯关系统设计文档

> 版本：v2.0 | 日期：2026-06-18

## 一、系统概述

### 1.1 核心设计
PVE 闯关模式采用**友方 vs 敌方**双阵营设计，无主忠反内身份划分。

- **胜利条件**：敌方全员阵亡 + 自己未死亡
- **失败条件**：自己死亡
- **先手规则**：每关指定出牌顺序（`turnOrder`），无 HP+1 增益

### 1.2 战场人数
支持 2-8 人场，动态座位布局：

| 人数 | 布局规则 | 有效座位索引 |
|------|----------|-------------|
| 8人 | 全满 | 0,1,2,3,4,5,6,7 |
| 7人 | 对角不填 | 0,1,2,3,5,6,7 |
| 6人 | 左右隔2位不填 | 0,1,2,3,5,7 |
| 5人 | 左右隔1位+对角不填 | 0,1,3,5,7 |
| 4人 | 左右隔1位+左右隔2位不填 | 0,3,5,7 |
| 3人 | 仅自己+左右隔2位 | 0,3,5 |
| 2人 | 仅自己+对角 | 0,4 |

（"左右隔2位" = 对角位置的左右侧）

### 1.3 AI 行为
所有非玩家角色由 AI 控制，使用 `DelayedAIDriver` 包装 `AIDriver`，行为逻辑与 PVP 模式中 AI 对手一致。

## 二、关卡数据结构

```typescript
interface PVELevel {
  id: number;           // 关卡ID（全局唯一，跨章节递增）
  name: string;         // 关卡名称
  enemyCount: number;   // 敌方人数
  allyCount: number;    // 我方人数（含玩家）
  enemyHeroes: string[]; // 敌方阵容（heroId数组）
  turnOrder: string[];  // 出牌顺序：'main','ally1','ally2',... 或敌将heroId
  bannedHeroes: string[]; // 禁选武将
  description?: string; // 关卡描述
}

interface PVEChapter {
  id: string;
  name: string;
  nameEn: string;
  region: string;        // 地区名（用于BGM/背景切换）
  cssClass: string;
  desc: string;
  requiredLevel: number; // 解锁所需等级
  levels: PVELevel[];
  placeholder?: boolean; // 是否为占位章节（未制作）
}
```

### 2.1 出牌顺序格式
`turnOrder` 数组定义出牌顺序：
- `'main'` — 玩家主将
- `'ally1'`, `'ally2'`, ... — 我方副将
- 敌将的 heroId（如 `'kaeya'`, `'diluc'`）

### 2.2 星级存储
- 星级记录保存在 `localStorage`，key = `genshin_card_pve_stars`
- 格式：`{ levelId: stars }`
- `savePVEStar`：仅当新星级高于旧星级时更新
- `getChapterStars`：章节累计星级
- `isChapterUnlocked`：等级 + 前一章节累计20星

## 三、章节架构

### 3.1 章节定义
共 9 章：蒙德→璃月→稻妻→须弥→枫丹→纳塔→挪德卡莱→至冬→深境螺旋

### 3.2 解锁条件
- **等级要求**：
  - 蒙德 Lv.1 | 璃月 Lv.4 | 稻妻 Lv.7 | 须弥 Lv.10
  - 枫丹 Lv.13 | 纳塔 Lv.16 | 挪德卡莱 Lv.21 | 至冬 Lv.26 | 深境螺旋 Lv.31
- **星级要求**：前一章节累计 ≥ 20 星
- **未制作章节**：显示"该地图暂未制作"
- **未解锁章节**：显示"等级/星级未满足解锁地图的需求"

### 3.3 BGM/背景
- PVE 模式：根据关卡所属章节的 `region` 设置背景和BGM（非英雄地区）
- 壁纸编号映射：蒙德 10-19 | 璃月 20-29 | 稻妻 30-39 | 须弥 40-49 | 枫丹 50-59 | 纳塔 60-69 | 挪德卡莱 70-79
- 深境螺旋暂用至冬的 BGM 和壁纸
- 每分钟随机切换壁纸（10张一组，Fisher-Yates洗牌）
- 平时BGM播放5分钟后切换战斗BGM

## 四、选将流程

### 4.1 流程
1. 关卡加载 → 获取 `allyCount`, `bannedHeroes`
2. 排除禁选池中的武将
3. **选主将**：展示候选池（排除禁选），玩家选1个
4. **选副将1**：刷新候选池（排除已选），玩家选1个
5. **选副将2**：重复，直到选满 `allyCount` 个
6. 敌方阵容由关卡数据预定义，无需选择

### 4.2 候选池规则
- 候选数量 = `3`（每次展示3个供选择）
- 每次选择后池子刷新，排除已选武将和禁选武将
- 七神武将也包含在候选池中

## 五、游戏逻辑

### 5.1 阵营系统
```typescript
enum Faction { Ally = 'Ally', Enemy = 'Enemy' }
interface PlayerState {
  faction: Faction;      // 阵营
  pveSeatIndex: number;  // PVE 座位索引
}
```

### 5.2 胜负判定
- 无身份系统 → `assignRoles` 被跳过
- 使用阵营判定替代身份判定
- `checkGameOver()` 逻辑：
  - 敌方全员阵亡 + 玩家存活 → **胜利**
  - 玩家阵亡 → **失败**

### 5.3 出牌顺序
使用 `turnOrder` 而非默认的顺时针顺序。`GameFlowController.startGame` 接受可选的 `turnOrder: number[]` 参数。

### 5.4 奖惩机制
- 击杀敌方无摸牌奖励
- 击杀友方副将无惩罚

### 5.5 星级评定
通关时根据友方存活人数：
- 全员存活 → ★★★（3星）
- 阵亡1人 → ★★（2星）
- 阵亡2人 → ★（1星）

## 六、座位布局实现

### 6.1 座位分配

```typescript
// finishPVEHeroSelect()
const seatLayout = this.getPVESeatLayout(totalPlayers);
for (let i = 0; i < seatLayout.length; i++) {
  this.pveSeatMap.set(seatLayout[i], i);
}
```

- 友方玩家先分配（playerId 0 ~ allyCount-1），敌方后分配
- 每个 PlayerState 的 `pveSeatIndex` 标记其在 8 人桌面上的座位

### 6.2 渲染布局

`getPVEBattlefieldLayout()` 以人类玩家座位为起点（右下），逆时针一圈：
- right 列：`hSeat+1`, `hSeat+2`（从下到上）
- top 行：`hSeat+5`, `hSeat+4`, `hSeat+3`（从左到右，逆时针远→近）
- left 列：`hSeat+6`, `hSeat+7`（从上到下）
