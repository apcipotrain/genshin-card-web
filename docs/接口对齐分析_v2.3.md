# PVP/PVE 接口对齐分析报告

> 版本：v2.3 | 日期：2026-06-30 | 状态：分析报告，暂不修改

---

## 一、PVP 与 PVE 接口差异总览

### 1.1 PlayerState 字段差异

| 字段 | PVP | PVE | 说明 |
|------|-----|-----|------|
| `role` | Monarch/Minister/Rebel/Traitor | `RoleType.None` | PVE 无身份系统 |
| `faction` | 无 | `'Ally'` / `'Enemy'` | PVE 阵营判定 |
| `pveSeatIndex` | 无 | `0-7` | PVE 动态座位 |
| `playerName` | 有（昵称） | 有（英雄名） | 显示一致 |

**状态**: ✅ 对齐。`isEnemy()` 优先检查 `faction`(PVE)，fallback 到 `role`(PVP)。

### 1.2 游戏初始化差异

| 方面 | PVP | PVE |
|------|-----|-----|
| 驱动 | `GameHost` (服务端) | `GameFlowController` 直接本地 |
| 玩家创建 | 服务端 `GameHost.initGame()` | 客户端 `makePVEPlayer()` |
| AI延迟 | 固定 1200ms | 用户可调 300ms-5000ms |
| turnDelay | 2500ms | 默认值 |
| BGM来源 | 主公`region` | 章节`region` |
| turnOrder | 无（顺时针） | 关卡`turnOrder`指定 |
| 身份分配 | `assignRoles()` 随机 | 无（全部 `RoleType.None`） |

**状态**: ✅ 对齐。两套初始化路径各司其职，通过同一套核心引擎运行。

### 2.2 PVP prompt 类型对齐

RemotePlayerDriver 通过 `prompt` 事件发送的请求类型需与客户端 `handlePVPPrompt` 完全匹配。

**已知问题** ⚠️ ：新增4位英雄引入了新的 prompt 方法，仅在 AIDriver 中实现，**未在 RemotePlayerDriver 和 HumanWebUIDriver 中实现**：

| 方法 | AIDriver | RemotePlayerDriver | HumanWebUIDriver | 影响 |
|------|----------|-------------------|------------------|------|
| `promptMagicGuess` | ✅ | ❌ 缺失 | ❌ 缺失 | 林尼魔术技能在PVP中无法工作 |
| `promptNaviaPersuadeMode` | ✅ | ❌ 缺失 | ❌ 缺失 | 娜维娅说服技能在PVP中无法工作 |

> **说明**: 代码使用 `?.` 可选链调用，缺失时返回 `undefined`，不会崩溃但技能无法正常执行。
> RemotePlayerDriver 需新增对应 `request` 类型，HumanWebUIDriver 需新增对应交互 UI。

## 三、存储接口对齐

### 3.1 服务端持久化（accounts.json）

| 数据 | 保存时机 | 加载时机 | 对齐状态 |
|------|----------|----------|----------|
| `level` / `exp` | 每次 `addExp` / `addExpByAccountId` | 服务端启动 `loadFromFile` | ✅ |
| `pveStars` | 客户端 `save_pve_stars` → `savePVEStars` | 服务端启动 `loadFromFile` | ✅ (v2.3 修复 sanitize) |
| `settings` | 客户端 `save_settings` → `saveSettings` | 服务端启动 `loadFromFile` | ✅ (v2.3 修复 sanitize) |
| `nickname` / `avatar` | `updateProfile` 即时保存 | 服务端启动 `loadFromFile` | ✅ |
| `createdAt` | 注册时保存 | 服务端启动 `loadFromFile` | ✅ |

**已修复 (v2.3)**: `sanitize()` 方法从前使用白名单方式，导致 `pveStars` 和 `settings` 被丢弃。已改为 spread 方式，保留所有字段（仅排除 `passwordHash`）。

### 3.2 客户端缓存（SettingsCache）

| 缓存键 | 存储方式 | 同步链路 | 对齐状态 |
|--------|----------|----------|----------|
| `bgmEnabled` | 内存 + 服务端 | `cacheSave` → `save_settings` → 服务端 → `profile_data` → `syncAllSettings` | ✅ |
| `voicelineEnabled` | 内存 + 服务端 | 同上 | ✅ |
| `bgmVolume` | 内存 + 服务端 | 同上 | ✅ |
| `voicelineVolume` | 内存 + 服务端 | 同上 | ✅ |
| `aiSpeed` | 内存 + 服务端 | 同上 | ✅ |

**注意** ⚠️：SettingsCache 纯内存存储，页面刷新后数据丢失。依赖服务端 `profile_data` 事件恢复。如果服务端未响应 `get_profile`（如离线状态），设置会回退到默认值。

**PVE 星级存储**：
```
PVELevels._pveStars (内存)
  → savePVEStar() → socketManager.emit('save_pve_stars')
  → 服务端 accountManager.savePVEStars()
  → accounts.json（持久化）
  
服务器重启后：
  → loadFromFile() 加载 accounts.json
  → login/get_profile → profile_data.account.pveStars
  → syncPVEStarsFromServer() → 恢复内存
```

**状态**: ✅ 修复后对齐。

## 四、经验系统接口对齐

### 4.1 计算逻辑一致性

| 模块 | PVP 服务端 | PVE 客户端 | 一致性 |
|------|------------|------------|--------|

### 4.2 PVP/PVE 经验上报

| 模式 | 计算方 | 上报方式 | 服务端处理 |

## 五、已知潜在问题汇总

### 5.1 高优先级 ⚠️

| # | 问题 | 影响范围 | 建议 |
|---|------|----------|------|
| 1 | `promptMagicGuess` 未在 RemotePlayerDriver 实现 | PVP 模式下林尼魔术技能无法执行 | 需在 RemotePlayerDriver 添加 `promptMagicGuess` 方法，发送 `prompt` 事件给客户端，客户端 HumanWebUIDriver 添加猜测 UI |
| 2 | `promptNaviaPersuadeMode` 未在 RemotePlayerDriver 实现 | PVP 模式下娜维娅说服技能无法执行 | 同上，添加模式选择 UI |

### 5.2 中优先级

| # | 问题 | 影响范围 | 建议 |
|---|------|----------|------|
| 3 | SettingsCache 无 localStorage 兜底 | 刷新页面且在服务器未响应前，设置回退默认值 | 可选：添加 `localStorage` 作为离线兜底，服务器数据优先 |
| 4 | `playerName` 字段在 PVP 中使用账户 `nickname`，在 PVE 中使用英雄 `name`，语义不一致 | 游戏记录中玩家标识不同 | PVE 可考虑显示 "nickname(heroName)" 格式 |
| 5 | `clorinde` 剧团技能在 PVP 中的 `RemotePlayerDriver` 需要知道当前是否禁技状态 | 影响技能按钮的可用性展示 | 需在下发 `game_event` 时包含剧团状态 |

### 5.3 低优先级

| # | 问题 | 影响范围 | 建议 |
|---|------|----------|------|
| 6 | `sigewinne` 的 `isSigewinneTempActive` 在 PVP 服务端的 SkillManager 接口需要同样声明 | PVP 桃使用时需检查温度技能 | PVP 服务端 `GameHost` 的 `skillManager` 接口已有此方法（通过 `as any` 绕过），但类型定义不完整 |
| 7 | PVE 经验值上报使用 `emitWithAck`，如果服务器离线则失败 | PVE 离线游玩经验不保存 | 可选：添加本地经验暂存 + 重试机制 |

## 六、接口对齐总评

| 维度 | 状态 | 说明 |
|------|------|------|
| Socket 事件 | ✅ 完全对齐 | 所有 C→S 和 S→C 事件均有对应处理 |
| PlayerState | ✅ 对齐 | 通过 `faction`(PVE) / `role`(PVP) 双轨判定 |
| 账号数据 | ✅ v2.3 修复 | `sanitize()` 已修复保留 `pveStars` 和 `settings` |
| 经验系统 | ✅ 对齐 | 服务端/客户端计算逻辑完全一致 |
| 设置同步 | ✅ 对齐 | 内存+服务端双存储，无 localStorage 污染 |
| PVP prompt 类型 | ⚠️ 部分缺失 | 2个新增 prompt 方法未在 RemotePlayerDriver 实现 |
| 新英雄 PVP 兼容 | ⚠️ 部分缺失 | 林尼、娜维娅技能在 PVP 中需要额外适配 |

**结论**: 核心接口已对齐，新增4位英雄在 PVE 中可正常运行。PVP 适配需要补全 `RemotePlayerDriver` 和 `HumanWebUIDriver` 中新增的 2 个 prompt 方法。
